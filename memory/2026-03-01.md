## Wingsuit Pitch Stability Tuning (sim branch → master)

### Center cell tuned values (locked in)
- cm_0 = 0.085, cm_alpha = -0.255, cp_0 = 0.36, cp_alpha = 0.025
- These values give trim ~90 mph, range ~55–115 mph
- Key insight: cm_alpha = -0.255 is the primary stabilizer (2.5× original)
- cp_0 near CG (0.36 vs CG 0.40) with positive cp_alpha = stabilizing CP travel

### Distributed pitch/CP to other segments
- Inner wing: cm_0=0.01, cm_alpha=-0.20, cp_0=0.15, cp_alpha=0.04
- Outer wing: cm_0=0.005, cm_alpha=-0.15, cp_0=0.18, cp_alpha=0.03
- Gradient: strongest center → weakest tips (matches real wingsuit physics)

### Gamepad pitch authority increased
- PITCH_ALPHA_MAX_DEG: 1.5 → 3.5 (α offset is transient — cm_alpha fights it back to trim)
- PITCH_CP_SHIFT: 0.05 → 0.13 (CP shift changes steady-state, unlike α offset)
- PITCH_CL_ALPHA_DELTA: 0.1 → 0.2
- PITCH_CD0_DELTA: 0.005 → 0.01
- Key insight from Hartman: α offset creates dynamic response but doesn't change steady-state trim much because cm_alpha compensates. CP shift actually moves the equilibrium point.

### Sim observations
- Roll throttle works very well — requires active piloting through turns, realistic feel
- Canopy model flies stable but needs CP/CM tuning like we did for wingsuit
- Dirty knob at ~0.23 works well for wingsuit, managed manually
