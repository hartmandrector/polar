## Sim velocity blue dot on speed polar
- Blue dot shows **actual** simulation velocity (ground speed + vertical speed) from body→inertial DCM rotation
- White dot = theoretical sustained speed at current α; blue dot = where vehicle actually is
- `SimRunner.inertialVelocity` getter: full 3-2-1 Euler DCM body→NED, then groundSpeed = √(Vn²+Ve²), verticalSpeed = Vd
- `setSimVelocity()` API on polar-charts, cleared on sim stop
- `simVelocityPlugin` renders blue filled dot with white border + "V: XX.X m/s" label
- Respects m/s vs mph toggle

## Key design notes
- Speed polar axis mapping: CD→Vys (drag=sink), CL→Vxs (lift=horizontal). Zero lift = Vxs=0 line, zero drag = Vys=0 line
- Negative CL (backflying) = negative Vxs, drag always positive = Vys always positive
- This mapping matters for GPS data interpretation — raw GPS gives inertial velocity which maps to the speed polar
